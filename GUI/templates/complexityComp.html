Run Time Comparison

RUNNING TIME ANALYSIS

The Jarvis March algorithm has a complexity of O(nh) which in the worst case is of the order O(n2). Whereas the KPS algorithm has a faster runtime complexity of O(nlogh) which in the worst case is of the order O(nlogn)

Overall:

The difference in time taken for the algorithms to complete their processing takes up to 62 times at the 1 million points mark. The difference starts out small but only gets more exaggerated as the number of points increases. This is to be expected as we can see from the graph as well, the difference in the rate of increase of O(n2) and O(nlogn).
