INTRODUCTION

A convex hull algorithm is a fundamental concept in computational geometry used to find the smallest convex polygon that encloses a set of points in a plane. It has numerous applications in various fields such as computer graphics, geographic information systems, robotics, and image processing.

In simple words, imagine you have a collection of nails on a flat surface. Now, take a rubber band and stretch it around the outermost nails, ensuring that it encloses all the nails within it and that it doesn't have any concavities or dents. The rubber band should be pulled tight so that it hugs the outer boundary of the nails. In this analogy, the nails represent points in a geometric space and the rubber band represents the convex hull of those points.

There are several famous algorithms which aim to compute the convex hull in the shortest period of time, such as the Graham Scan, Chan's Algorithm, Jarvis March, Kirkpatrick Seidel, and many more.

The visualization of these convex hull computing algorithms serves to analyse and explain the beauty behind complex calculations carried out by a computer program. Being able to see how the algorithm works step by step promotes the understanding of an algorithm, perhaps also providing a guiding light towards a more efficient algorithm.

For the purpose of this assignment we have implemented the visualization of:

  • Jarvis March: An algorithm that repeatedly finds the point with the smallest polar angle relative to the current point and adds it to the convex hull.
  • Kirkpatrick-Seidel algorithm: This algorithm iteratively divides the point set into smaller subsets, constructing convex layers of increasing granularity until the convex hull is formed. It leverages the concept of extreme points and triangulation to efficiently compute the convex hull.
