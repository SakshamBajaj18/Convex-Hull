JARVIS MARCH ALGORITHM

RUNNING TIME ANALYSIS

The time complexity of Jarvis's march algorithm is O(nh), where n is the total number of points and h is the number of points on the convex hull. This means the running time depends on both the number of input points and the number of points in the final hull. In the best case scenario, where the number of hull points is small compared to the total number of points, the algorithm performs well. However, in the worst case scenario, where all points are on the hull, the time complexity becomes O(n2). Thus it can be summarized as follows:

  • Time Complexity = O(nh), where n: total number of points, h: number of points on the convex hull
  • Best Case Time Complexity = O(n), where h << n

BREAKING DOWN THE TIME COMPLEXITY

The Jarvis March algorithm has a time complexity of O(nh) where:

  • n is the total number of input points.
  • h is the number of points on the convex hull.

Breakdown of Factors:

Here's a breakdown of the factors contributing to the time complexity:

  • Outer Loop:

    This loop iterates until the `current` point becomes the starting point again, effectively traversing the convex hull boundary once. In the worst case, this loop runs for h iterations.

  • Inner Loop:

    Within each outer loop iteration, this loop iterates through all points except the current one (n-1 points). Inside the inner loop, the `checkCounterClockWise` function is called, which itself has a constant time complexity (O(1)). The total number of times the inner loop runs is (n-1) * h .

Early Termination:

    The inner loop might terminate early if a suitable next point is found before iterating through all remaining points. This can significantly improve the average-case performance, especially for datasets where the convex hull is a small fraction of the total points.

Overall:

In the worst case where the convex hull is large compared to the total points or early termination rarely occurs, the time complexity remains O(nh). On average, for datasets with a smaller convex hull or frequent early termination, the time complexity could be closer to O(n log h) or even better in some scenarios.
